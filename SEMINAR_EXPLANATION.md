# AI・ロボットで実現する省力化セミナー アンケートアプリ 開発プロセス

## このアプリを作った目的

セミナー会場で参加者にスマートフォンからアンケートに回答してもらい、
その結果をリアルタイムで会場スクリーンにグラフ表示するWebアプリです。

- 参加者はQRコードを読むだけで、すぐに回答画面が開く
- 氏名・メールアドレスなどの登録は一切不要（匿名回答）
- 回答するたびにグラフがリアルタイムで更新される

## AIが自律的に行ったこと

### Step 1: 実装担当（Opus）が構築した14ファイル

| ファイル | 役割 |
|---------|------|
| server.js | バックエンドサーバー（API + WebSocket） |
| package.json | プロジェクト定義 |
| index.html + survey.css + survey.js | 参加者の回答画面（モバイル対応） |
| admin.html + admin.css + admin.js | 管理者ダッシュボード |
| projection.html + projection.css + projection.js | 会場投影スクリーン |
| common.css | 共通デザイン基盤 |
| README.md | ドキュメント |
| .gitignore | Git管理対象外の定義 |

### Step 2: デザイン担当（Sonnet）が改善した内容

- カラーパレットの再設計（原色を排除、調和のとれた配色に）
- アニメーションの品質向上（カードの出現演出、ボタンの光沢効果）
- 投影画面の視認性強化（フォントサイズ拡大、背景グロー効果追加）
- タップ領域の拡大（スマホ操作性向上）
- アクセシビリティ対応（スクリーンリーダー、コントラスト比）

### Step 3: QA担当（Sonnet）が発見・修正した問題

- **重大なバグ 0件**（動作不能レベルの問題なし）
- **中程度の問題 3件**（すべて修正済み）
  - 投影画面のリセット時に画面が戻らない問題
  - 特定条件で結果が表示されない問題
  - メモリリークの可能性
- **軽微な問題 4件**（すべて修正済み）
  - プライベートブラウズ対応
  - エラー処理の強化
  - HTML構造の改善

## 使用した技術スタック

| カテゴリ | 技術 |
|---------|------|
| バックエンド | Node.js + Express |
| リアルタイム通信 | Socket.IO（WebSocket） |
| フロントエンド | HTML5 / CSS3 / JavaScript（フレームワークなし） |
| グラフ描画 | Chart.js |
| QRコード生成 | qrcode（サーバーサイド） |
| ホスティング | Render（GitHub連携で自動デプロイ） |
| ソースコード管理 | GitHub |
