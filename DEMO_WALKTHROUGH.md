# セミナーデモ：Claude Codeで40分でWebアプリを開発する全手順

## 前提条件

- Windows 11 PC
- Node.js インストール済み
- Git インストール済み
- GitHubアカウント作成済み
- GitHub CLI（gh）インストール済み

---

## Phase 1: 環境構築

### 1-1. VS Codeをインストール
- https://code.visualstudio.com/ からダウンロード・インストール

### 1-2. VS Code拡張機能「Claude Code」をインストール
- VS Codeを開く
- 左サイドバーの拡張機能アイコン（四角が4つ）をクリック
- 検索欄に「Claude Code」と入力
- Anthropic公式の「Claude Code」を「Install」

### 1-3. Claude Codeを起動
- VS Codeのサイドバーにクロードのアイコンが追加される
- クリックしてClaude Codeパネルを開く
- Anthropicアカウントでログイン（初回のみ）

### 1-4. 作業ディレクトリを準備
- VS Codeでフォルダを開く（例: `c:\work\Antigravity`）
- Claude Codeのチャット欄から指示を出す準備完了

---

## Phase 2: 要件を伝える

### 2-1. アプリの目的を伝える

Claude Codeのチャット欄に以下のような指示を入力:

```
セミナー会場向けのリアルタイム集計アンケートWebアプリを実装したい。

製造業向けセミナーの冒頭で、参加者にスマホから5問の選択式アンケートへ
匿名回答してもらい、会場のスクリーンにリアルタイムでグラフ表示したい。
```

### 2-2. 利用フローを伝える

```
・講師がQRコードを表示する
・参加者がスマホでQRコードを読む
・すぐに回答画面が開く（ログイン不要）
・回答後、グラフがリアルタイム更新される
```

### 2-3. 質問内容を伝える

```
Q1: 貴社で現在使っている生成AIを選んでください（複数選択可）
選択肢: Copilot / Gemini / ChatGPT / Claude / その他 / 使っていない

Q2: 貴社で生成AIをどのように使っていますか（単一選択）
選択肢: 全社で使っている / 一部の部署で使っている / ...

（以下、Q3〜Q5も同様に記載）
```

---

## Phase 3: 開発チームの構成を議論

### 3-1. チーム構成を相談

```
エージェントチームで実現するためのプロンプトを作成してください
```

AIが5エージェント並列構成を提案。

### 3-2. チーム構成を議論・改善

```
リーダーやチェックをする担当、デザインのスキルを持った担当は必要ありませんか。
また、開発は1人でできませんか。メリット・デメリットを含めて回答してください。
```

AIがパターン比較（1人 / 5人並列 / 役割分担あり）を提示。

### 3-3. 最終構成を決定

```
実装とチェック、デザインは役割を専門家に任せたいです。リーダーも必要では？
```

最終的に4名体制に決定:
- **リーダー**: 人間 + メインAI
- **実装担当**: Claude Opus（最上位モデル）
- **デザイン担当**: Claude Sonnet（中位モデル）
- **QA担当**: Claude Sonnet（中位モデル）

### 3-4. AIモデルの選定理由

```
それぞれのAIモデルは何を使いますか？適切なレベルのものを選択してください。
```

- Opus: 14ファイルを一気に構築する複雑なタスクに最適
- Sonnet: 明確な基準に沿ってレビュー・改善するタスクに十分
- Haiku: 今回は不採用（品質に直結する作業のため速度より精度を優先）

---

## Phase 4: 開発実行

### 4-1. 開発開始を指示

```
開始してください
```

この一言で、AIが以下を自動実行:

### Step 1: 実装担当エージェント（Opus）

全14ファイルを約8分で構築:
- server.js（バックエンドAPI + WebSocket）
- 参加者回答ページ（モバイル対応HTML/CSS/JS）
- 管理者ダッシュボード（Chart.jsグラフ）
- 会場投影スクリーン（ダークテーマ）
- package.json、README.md、.gitignore

リーダー（AI）が完了確認: サーバー起動テスト → OK

### Step 2: デザイン担当エージェント（Sonnet）

CSSを専門的にレビュー・改善（約7分）:
- カラーパレット再設計（原色を排除、プレミアムな配色）
- アニメーション品質向上（カードの出現演出、ボタンの光沢効果）
- 投影画面の視認性強化（フォントサイズ拡大、グロー効果）
- タップ領域拡大（スマホ操作性向上）
- アクセシビリティ対応

リーダー（AI）が改善内容を確認 → OK

### Step 3: QA担当エージェント（Sonnet）

全コードを第三者目線でレビュー（約3分）:
- API整合性チェック（サーバーとフロントのURL・データ構造一致確認）
- バリデーション確認
- WebSocket接続確認
- セキュリティチェック（XSS対策）
- 発見した問題7件をすべて修正

結果: Critical 0件 / Major 3件（修正済） / Minor 4件（修正済）

## Phase 6: インターネット公開（Render）

### 6-1. 公開URLを要求

```
ローカルホストではなく、公開URLを使ってください
```

### 6-2. ホスティングサービスの選定

AIがRender vs Vercelの比較を提示:
- Vercel → WebSocket非対応のため不可
- Render → WebSocket対応、無料プランあり → 採用

### 6-3. Renderにデプロイ

1. render.yaml（設定ファイル）をAIが自動生成・プッシュ
2. https://render.com でGitHubアカウント連携
3. リポジトリを選択してデプロイ
4. 公開URL発行: https://seminar-survey.onrender.com/
